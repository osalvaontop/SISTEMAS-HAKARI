🎯 FUNCIONALIDADES IMPLEMENTADAS
═══════════════════════════════════════════════════════════════════════════

  ✅ /painelcall
     → Envia embed com botão "Criar Call" no canal 1507595486889115698
     → Apenas administradores podem usar
     → Outras pessoas clicam para criar suas calls

  ✅ /configcall
     → Abre menu com 3 botões de configuração
     → Apenas o criador da call pode usar
     → Botões: "Nome", "Limite", "Pública"

  ✅ Botão "Nome"
     → Abre modal para digitar o nome
     → Se vazio, volta ao padrão "Call de <username>"
     → Máximo de 100 caracteres

  ✅ Botão "Limite"
     → Abre modal para digitar um número
     → Se vazio ou 0, a call fica sem limite
     → Aplica limite de pessoas na call

  ✅ Botão "Pública"
     → Alterna entre verde (🟢 Pública) e vermelho (🔴 Privada)
     → Pública: Qualquer um pode entrar
     → Privada: Apenas o criador pode entrar
     → Permissões aplicadas automaticamente

  ✅ /deletarcall
     → Deleta o canal de voz da call
     → Remove todos os dados
     → Apenas o criador pode usar

  ✅ /listarcalls
     → Lista todas as calls ativas
     → Mostra: criador, nome, limite, status, pessoas
     → Apenas administradores podem usar

  ✅ Persistência de Dados
     → Todos os dados salvos em calls_data.json
     → Dados persistem entre reinicializações
     → Carregamento automático

🐛 PROBLEMAS COMUNS
═══════════════════════════════════════════════════════════════════════════

  ❌ "Slash command não aparece"
     → Reinicie o bot, aguarde 30s

  ❌ "Você já possui uma call ativa!"
     → Execute /deletarcall antes de criar outra

  ❌ "Botões não funcionam"
     → Reinicie o bot

  ❌ "Canal não é criado"
     → Verifique permissão "Gerenciar Canais"

  ❌ "Erro ao conectar na call privada"
     → Apenas o criador pode entrar em calls privadas
